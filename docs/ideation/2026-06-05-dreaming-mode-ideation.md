# Ideation: Dreaming Mode for Hypermnesic

> Status: **ideation only** — directions worth exploring, not a plan or requirements.
> Next step for any idea: `/ce-brainstorm` (which `/feature-dev` and `/ce-plan` feed from).
> Generated by `ce-ideate` run `d4ea3c01`. Scratch artifacts (grounding, 48 raw candidates,
> survivors) under `/tmp/compound-engineering/ce-ideate/d4ea3c01/`.

## Framing

"Dreaming" = a background process that runs when the system is idle and transforms raw
captures into higher-value durable knowledge: consolidation (episodic→semantic),
compression, relationship discovery, contradiction resolution, importance re-ranking, and
abstraction/concept formation.

**The decisive insight from grounding:** for a files-are-source-of-truth engine, the safest,
most defensible "dream" looks *different* from Anthropic's just-launched **Dreaming**
(2026-05-06) or Honcho's `schedule_dream`. The skeptic literature's central finding
(Zhang et al., *"Useful Memories Become Faulty When Continuously Updated by LLMs"*,
arXiv 2605.12978 — GPT fails 46–54% of previously-solved problems after consolidating even
from ground truth) is that consolidation is "an iterated generative loop with no anchor."
**Git is that anchor.** hypermnesic's prime invariant ("files are truth; the index is a
disposable projection; a reindex must never lose a committed write") is precisely what every
LLM-memory consolidator lacks. So the whole feature reframes from *"synthesize prose"* to
*"propose additive structure over an immutable episodic ledger, and prove it never degrades
recall."*

## Grounding Context (Codebase)

- **Prime invariant:** durable dream output MUST be a committed markdown file, never an index
  mutation. The one write path is `commit_note.py` (git-first: guard → flock → fetch/ff +
  HEAD preflight → diff-or-die frontmatter gate → write → commit → push → index upsert →
  audit). Refusals return `{committed:false, refused}` — never silent success.
- **Zero generative-write path exists today.** The only OpenAI calls are embeddings
  (`embed.py`) and optional query-expansion (`expand.py`, gpt-4o-mini). Every "generated"
  artifact (salience digest, connection suggestions, daily review) is **deterministic
  template rendering**. A dream that writes synthesized prose would be the *first* generative
  write path — and any LLM step must be injectable + graceful-degrading (tests run offline,
  dense degrades to lexical).
- **Existing primitives to compose with, NOT reinvent:** `converge.py` (the canonical
  background-pass discipline — debounced, flock-safe, bounded budget, oversized-delta guard,
  OOM scar; writes only `.hypermnesic/`), `salience.py` (deterministic importance =
  link-degree + embedding-centrality + write-recency; **never** writes a score into a source
  note), `connect.py` (relationship discovery = high cosine + no existing wikilink, capped),
  `propose.py` (the one review-gated PR front door; per-day ProposalBudget caps flooding),
  `generated.py` (provenance marker + managed block), `daily_review.py` (closest existing
  periodic loop; manual CLI).
- **Binding doctrine:** "NOT an autonomous re-organizer — no silent file rewrites, EVER."
  "Consolidation is explicit and preserves raw evidence" (cite raw episodes, never overwrite;
  `sources/` is an immutable append-zone). No scores in source frontmatter (KTD2).
  Behavioural/preference/session memory belongs in **Honcho**, not hypermnesic.
- **Gaps a dream feature must fill:** (1) no scheduler/idle host [biggest]; (2) no
  generative-write path; (3) no episodic→semantic promotion; (4) no contradiction/temporal-
  invalidation primitive (Phase-3-deferred); (5) no abstraction/clustering primitive;
  (6) no rich "synthesized-from-[X,Y,Z]" provenance schema; (7) no access/read-recency
  signal; (8) no threat model for an autonomous LLM write surface (CODEOWNERS-routed files).

### External prior art (the competitive & safety frame)

- **Anthropic "Dreaming"** (launched 2026-05-06): scheduled background pattern-extraction over
  past sessions/memory; weights frozen ("learning to write better notes for their future
  self"); additive (separate store, input untouched); auto vs review-before-write modes.
- **Honcho `schedule_dream`:** two phases — Deduction (contradiction resolution, supersession)
  then Induction (cross-conclusion patterns; require ≥2 sources; confidence scaled by support).
  Scheduling: ≥50 new items + ≥8h cooldown + 60-min idle timer that cancels on activity.
  Failure mode (issue #608): silent dreamer death → need a `last_dream` health signal.
- **Letta sleep-time compute** (git-backed MemFS markdown), **Stanford Generative Agents**
  (reflection trees fired on accumulated-importance threshold; reflections retrieved
  alongside observations), **Zep/Graphiti** (bi-temporal edges + LLM contradiction →
  invalidate-not-delete), **Mem0** (pivoted to ADD-only — market signal against destructive
  consolidation), **HippoRAG** (PageRank over the graph beats RAG on multi-hop ~20% — graph
  *density* drives recall), **A-MEM** (Zettelkasten linking).
- **Git-native memory ecosystem (hypermnesic's home turf):** Karpathy's LLM Wiki ("stop
  re-deriving, start compiling"; lint for contradictions/stale/orphans) + LLM Wiki v2
  (confidence decay, supersession = "version control for knowledge", forgetting =
  deprioritization); gitmem (literal "Dream pipeline" governed by **pull requests**);
  PIVOTEX (`dreams/` quarantine the user promotes/rejects); DiffMem (git-as-temporal-memory,
  grep/log/diff/blame retrieval); `ak dream` (Claim/Evidence/Compiled-Truth, vector-free).
- **Safety envelope:** snapshot→eval(before+after)→auto-rollback if recall regresses; cap
  proposals (~7/dream) for tractable review; belief-aware gating (human-confirmed memories
  read-only to the loop — "a loop that can degrade its own best work is worse than no loop");
  "Honest Lying" (open-ended self-reflection 0% vs programmatic feedback 86%).
- **Readwise library:** thin on memory/cognition topics — no usable user-frame to align to
  (honest gap).

## Topic Axes

1. Trigger & idle scheduling — when/where the dream runs given no in-repo scheduler.
2. Consolidation operations — the cognitive transforms (promotion, compression, linking, contradiction, abstraction, re-ranking).
3. Output & write governance — how dream output lands in the git vault (propose/quarantine, provenance/bi-temporal schema, additive-only).
4. Safety, drift-control & evals — not degrading memory (eval/rollback, belief-aware gating, the faulty-memory risk).
5. Review & human surface — how the user experiences/reviews/approves dreams.

The survivor set is also a coherent architecture: **#1** = where/when it runs · **#2** = who
writes · **#3 + #4** = what it does · **#5** = what the output carries · **#6** = why it's
safe · **#7** = how you see it.

## Ranked Ideas

### 1. Convergence-tier dreaming — no new daemon
**Description:** Ship `hypermnesic dream` as a CLI driven by an *external* timer (Hermes/
systemd, mirroring `daily_review`), reusing `converge.py`'s hardened idle discipline (flock,
debounce, bounded budget, oversized-delta guard) and gating the run on git-commit-rhythm idle
detection — cancel if the user is actively committing. A `last_dream` watermark (commit SHA)
is both the incremental cursor (`git log <SHA>..HEAD` = the free delta) and the silent-death
health beacon. Cadence tiers: cheap per-delta nightly + a rare deep whole-vault pass.
**Axis:** Trigger & idle scheduling
**Basis:** `direct:` — biggest stated gap is "no scheduler/idle host"; `converge.py` is "THE
canonical background-pass discipline"; `daily_review` is the existing CLI-periodic template.
`external:` Honcho's idle-timer-that-cancels-on-activity + ≥N/cooldown thresholds.
**Rationale:** Attacks the #1 infrastructure gap by *not* building a daemon — inherits
battle-tested concurrency/budget guards (native-primitives-first). The watermark closes
Honcho's documented "silent dreamer death" failure for free.
**Downsides:** Relies on an external scheduler (not self-contained); idle heuristics need
tuning per deployment (single-Mac vs org-server).
**Confidence:** 90% · **Complexity:** Medium · **Status:** Unexplored

### 2. Deterministic engine, client-side synthesis — the "dream brief"
**Description:** The engine never gains a generative-write path. A new read-only `dream` MCP
tool deterministically assembles a *brief* — surprisal-ranked deltas, contradiction
candidates, embedding clusters, orphans — mined from git history + `note_vectors()` +
`connect`/`salience`. The **client agent** (Claude Code/Codex) performs the LLM synthesis in
its own context, and any durable output returns through the *existing* `commit_note` write
scope.
**Axis:** Output & write governance (also touches Trigger & scheduling)
**Basis:** `direct:` — "ZERO generative-LLM-to-disk path exists today"; no threat model for an
autonomous LLM write surface (gap #8); MCP = 5 read + 1 gated write. `external:`
Anthropic/Letta both synthesize memory in the agent's context.
**Rationale:** Dissolves gaps #2 and #8 simultaneously — the engine keeps its deterministic-
only invariant and offline-green test suite; no new CODEOWNERS-routed write surface; the
client supplies the model. Cleanest fit to the repo's deepest architecture.
**Downsides:** Pushes orchestration onto the client; "dreaming" becomes a collaboration, not a
fully autonomous server feature (may feel less magical); needs a clean brief schema.
**Confidence:** 82% · **Complexity:** Medium · *(highest novelty)* · **Status:** Unexplored

### 3. Synthesis & linking ops — surprisal-prioritized, additive
**Description:** The flagship "what dreaming *does*": (a) **episodic→semantic promotion** of
the capture backlog — cluster aged `sources/` captures, propose a synthesized note that
*cites* them, never touching the immutable append-zone; (b) **concept/MOC abstraction** over
`note_vectors()` clusters (the missing clustering primitive); (c) **dedup/compression** within
clusters; (d) **orphan-rescue / missing-wikilink densification** (each added edge is a future
HippoRAG PageRank edge). Bounded budget spent surprisal-first (git-churn as free TD-error).
**Axis:** Consolidation operations
**Basis:** `direct:` gaps #3 & #5; `connect.py`/`salience.py` already compute candidates but
nothing acts on them. `external:` HippoRAG ("graph density drives recall ~20%"), A-MEM,
archival appraisal (retain-raw vs promote vs archive).
**Rationale:** Covers four of the six named capabilities (consolidation, compression,
relationship discovery, abstraction) on top of existing rails; closes the loop on primitives
that currently compute-but-don't-act.
**Downsides:** First generative-write path — the riskiest per the faulty-memory paper; depends
on #5 (provenance) and #6 (eval) to be safe; promotion quality is hard.
**Confidence:** 80% · **Complexity:** High · **Status:** Unexplored

### 4. Contradiction → bi-temporal supersession + blast-radius tracing
**Description:** Detect conflicting current-state claims (reuse `connect.py` cosine + an
*injectable* LLM judge). Instead of editing either note, **propose** bi-temporal supersession
(`valid_until`/`superseded_at` on the older claim, forward-link to the newer) and
**contact-trace** (epidemiology analogy) the downstream notes that inherited the now-stale
premise via the wikilink/`synthesized_from` graph, flagging the whole blast radius for review.
Raw claim preserved, marked stale, never deleted.
**Axis:** Consolidation operations
**Basis:** `external:` Zep/Graphiti bi-temporal edges + LLM-Wiki-v2 supersession. `direct:`
gap #4 (contradiction/temporal-invalidation is Phase-3-deferred, unbuilt).
**Rationale:** Pairwise contradiction detection fixes the edge but leaves silently-corrupted
downstream syntheses — exactly the memory-rot the doctrine warns about. Tracing the exposure
network turns one detection into a complete reviewable cleanup, without touching a source file.
**Downsides:** LLM contradiction judgment is error-prone; blast-radius BFS can over-flag;
touches the CODEOWNERS-routed frontmatter schema (security-sensitive, doc-heavy).
**Confidence:** 72% · **Complexity:** High · **Status:** Unexplored

### 5. Provenance + temporal schema, trial-balance gated — the one-time substrate
**Description:** Define *once*: `valid_from`/`valid_until` + `recorded_at`/`superseded_at` +
`synthesized_from:[slugs]` + run-identity (seed + base SHA + model). This schema permanently
unlocks contradiction-tracking, confidence-decay, "what did I believe in March?" time-travel,
and replayable/diffable dreams. A deterministic **"trial-balance" gate** (double-entry
analogy) refuses any synthesized claim lacking a balancing source citation (≥2 sources) — and
runs even when the LLM is neutralized offline.
**Axis:** Output & write governance
**Basis:** `external:` Graphiti bi-temporal, agent-knowledge active/superseded/disputed,
accounting integrity. `direct:` gaps #4 & #6; must land through the diff-or-die gate as
explicitly-set keys only.
**Rationale:** The highest-leverage *one-time* investment — getting it right in v1 is
load-bearing because retrofitting later means rewriting every dream note. The trial-balance
gate makes the first generative path auditable *by construction*, structurally preventing the
unanchored loop the faulty-memory paper condemns.
**Downsides:** Schema design is hard to reverse; risks over-engineering v1; frontmatter churn
must respect the 11%-abort surgical-edit discipline.
**Confidence:** 85% · **Complexity:** Medium · **Status:** Unexplored

### 6. Provably-safe by construction — eval gate + belief-aware fence + restraint-first
**Description:** Three composable guarantees: (a) **eval gate** — snapshot → run a held-out
recall eval (built on the existing `harness/`, offline) → *auto-reject* (never auto-apply) any
proposal batch that drops recall below baseline−threshold, so the user never sees a degrading
consolidation; (b) **belief-aware fence** — human-authored/confirmed notes (git-blame
authorship = "self," immune-system analogy) are read-only to the loop; only generated notes
are eligible; (c) **restraint-first** — the first dream op is *subtractive*: demote stale
**derived** notes (Ebbinghaus decay), never raw evidence.
**Axis:** Safety, drift-control & evals
**Basis:** `external:` Zhang et al. faulty-memory (episodic-only beat every consolidator),
Belief-Aware Memory, the operational playbook. `direct:` `harness/` exists; offline-
deterministic tests; "NOT an autonomous re-organizer — no silent rewrites EVER."
**Rationale:** Directly answers the governing product doctrine, and produces the
differentiating public claim Anthropic's closed Dreaming can't make: *"consolidation that
provably cannot lose a committed write or degrade recall — with held-out numbers."*
**Downsides:** Building a trustworthy held-out eval set is genuinely hard; thresholds are
judgment calls; belief-aware tagging needs a clear "confirmed" definition.
**Confidence:** 88% · **Complexity:** Medium-High · **Status:** Unexplored

### 7. The Dream Journal — one reviewable, recursively-reflectable digest
**Description:** Each run emits **one** dated digest (via `propose`/PR or `daily-review`) of
everything it proposed, the sources each was synthesized from, what it deferred, and open
questions it couldn't resolve — capped (~7 items) so review is one reading session with
accept/reject per item. The journal is itself a semantic note future dreams reflect on
(spotting recurring patterns — Stanford reflection trees), and its success KPI is the review
queue *shrinking*. v0 posture: ephemeral pull-only suggestions in `.hypermnesic/` behind a
read-only MCP tool, *before* any generative-write path exists.
**Axis:** Review & human surface
**Basis:** `direct:` `daily_review` (closest existing loop), `propose` ProposalBudget (R-1
flood risk), `generated.py` markers. `external:` Anthropic review-before-write, gitmem "review
what your agents learn like code."
**Rationale:** Review fatigue is what kills wikis ("maintenance grows faster than value"). A
single provenance-rich digest turns review from N scattered PRs into one coherent read — and
the recursively-reflectable journal makes each dream improve the next.
**Downsides:** Digest UX/format is product-shaping (gift vs chore); one-PR-many-items vs
many-PRs is an open call; the recursive-reflection loop needs its own drift guard.
**Confidence:** 84% · **Complexity:** Low-Medium · **Status:** Unexplored

# Integrated Spine: Articulating #1 + #5 + #7 Together

> Added 2026-06-05 (`ce-ideate` run `d5fb27a2`, same session). A focused follow-up that
> articulates how the **convergence-tier trigger (#1)**, the **provenance + bi-temporal

## The core finding

Across six independent framing lenses, the dominant result was the same: **#1, #5, and #7 are
not three features — they are three projections of one atomic record.** A dream is **one atomic
git commit** whose *frontmatter is the provenance* (#5), whose *commit SHA is the `last_dream`
watermark + health beacon* (#1), and whose *body is the journal entry* (#7). There is no
separate watermark file to drift; "did the dream run?", "what did it produce, from what?", and
"where do I review it?" all resolve to one record, keyed by `run-id`.

**One-line answer:** *#1+#5+#7 together = "the Close" — one atomic, deterministic, git-native
dream-commit per rested period, whose frontmatter is its provenance, whose SHA is its cursor,
and whose body is its reviewable journal entry. A lab notebook, not an autosave.*

These six lenses cohere: **#A** is what the record *is* · **#B** is what to *call* it · **#C**
is the *loop* · **#D** is the *boundary* · **#E** is the *extensibility* · **#F** is the *pitch*.

## Articulation lenses

### A. One Run-Record, three projections — the atomic "dream commit"
**Axis:** Shared data contract
**Description:** A dream is one atomic git commit. Frontmatter = the #5 schema (`valid_from`/
`valid_until`, `recorded_at`/`superseded_at`, `synthesized_from:[slugs]`, run-identity
`seed+SHA+model`); commit SHA = the #1 `last_dream` watermark + silent-death beacon; body = the
#7 journal entry. The latest journal entry's run-identity, read back, IS the cursor — no
separate dotfile. Keyed by `run-id` → (cursor, provenance, journal). Deferred/open items carry
`synthesized_from` SHA pointers so they survive the cursor advance and re-enter the next delta.
**Basis:** `direct:` prime invariant (durable run-state must be a committed file, not
disposable `.hypermnesic/`); #1's watermark-SHA and #5's run-identity-SHA are the same SHA
stated twice. `external:` git commit atomicity; LSM-tree WAL+compaction.
**Rationale:** Removes the worst seam — three drift-prone homes for run-state collapse to one
file that cannot disagree with itself; reindex cannot lose the cursor.
**Downsides:** Commits to "journal entries are durable" early (front-loads write-governance);
the deferral-pointer mechanism needs care.
**Confidence:** 90% · **Complexity:** Medium · **Status:** Unexplored

### B. Name it "the Close" — the nightly-close metaphor
**Axis:** Unifying frame & name
**Description:** Accounting's nightly close maps ~1:1: posting period = cadence (#1),
double-entry + trial balance = the ≥2-source gate (#5), general journal of record = the Dream
Journal (#7); "reversing entries, never erasing" = additive-only. Supporting doc lenses:
WAL-checkpoint (atomicity), MVCC tuples (bi-temporal, never-update-in-place), aircraft
maintenance logbook (counter-signed record; missing stamp = grounded).
**Basis:** `reasoned:` posting-period→cadence, trial-balance→2-source gate, journal-of-record→
Dream Journal; grounding flagged accounting as mapping "almost 1:1."
**Rationale:** "Every night, hypermnesic closes the books on what it learned" lands cadence +
balanced provenance + reviewable journal as one ritual; a close that doesn't balance posts
nothing — exactly the gate behavior.
**Downsides:** "Close" may read dry vs "dreaming"; a metaphor isn't load-bearing.
**Confidence:** 78% · **Complexity:** Low · **Status:** Unexplored

### C. The run loop in one sentence / one diagram
**Axis:** Run loop / lifecycle
**Description:** Onboarding + README artifact. Sentence: *"On a rested vault, the dream replays
what changed since `last_dream`, keeps only facts that ≥2 sources independently support, emits
one dated accept/reject journal entry, then advances the cursor — proposing, never rewriting."*
Pipe: `[HEAD − last_dream = delta] →idle-gate→ [candidates] →trial-balance ≥2 src→ [journal
entry, cap ~7] →accept→ [advance last_dream]`. One token (`run-id`) threads all stages. Atomic:
a half-run commits nothing — no "ran but produced nothing" path.
**Basis:** `direct:` #5 deterministic gate (offline), #7 cap+accept/reject, #1 watermark
advances on completion; `commit_note` refusals are `{committed:false}` not silent success.
`external:` Zhang et al./"Honest Lying" — the gate is the anti-hallucination anchor.
**Rationale:** If a contributor can't draw the spine in one pipe, it isn't coherent yet; also
the competitive one-liner (every arrow deterministic + human-gated).
**Downsides:** A clean diagram can hide real complexity (idle detection, deferral re-entry).
**Confidence:** 85% · **Complexity:** Low · **Status:** Unexplored

### D. Deterministic spine, the LLM rides later — the v0 boundary
**Axis:** Boundaries & invariants
**Description:** The whole spine is deterministic + offline-CI-green + write-confined: every
transition (idle-gate → delta → stamp → trial-balance → journal → advance) is pure/offline; the
generative LLM is an injectable later decorator (mirrors `expand.py`). The trial-balance gate is
the *promotion license* from ephemeral `.hypermnesic/` → durable git. Ship #7 read-only first
(no generative-write path crossed); #1 and #5 clip on as pure additions. `git diff` on source
notes after a dream must be empty.
**Basis:** `direct:` tests run offline (key neutralized, dense→lexical); "ZERO generative-LLM-
to-disk path today"; gap #8 (no threat model for an autonomous write surface); salience never
writes a score into a source note.
**Rationale:** Makes v0 shippable under all six CI gates today with zero new generative-write
risk; turns the LLM into a later isolated PR; draws the exact line where the write path will
later be added (through `propose`, deliberately).
**Downsides:** A deterministic-only v0 may underwhelm "AI dreaming" expectations; the
promotion-license design touches CODEOWNERS-routed files.
**Confidence:** 88% · **Complexity:** Medium · **Status:** Unexplored

### E. The Dream Runtime — three sockets every future op inherits
**Axis:** Run loop / leverage
**Description:** Articulate the union as a runtime, not three features: #1 = scheduler+budget
socket, #5 = output-contract socket (every op returns stamped, trial-balance-passing
proposals), #7 = review socket (every op renders in the same journal). A future op (#3
synthesis, #4 contradiction, #6 eval) is just `op(brief) -> [stamped_proposal]` — zero new
scheduling/schema/review code. The watermark+provenance+journal triple is simultaneously a
replayable audit ledger of self-modification.
**Basis:** `direct:` `converge.py` budget/flock discipline + `propose.py` as the one
review-gated front door + `generated.py` uniform marker. `external:` Honcho/Letta dual-process;
gitmem "review what your agents learn like code."
**Rationale:** The strongest argument for building these three first — the spine buys down the
cost of every later op and converts N future safety reviews into one. One payload, three
consumers, zero special-casing.
**Downsides:** "Runtime/SDK" risks over-abstraction before a second op validates the interface
(YAGNI); the `brief` contract is a hard up-front design.
**Confidence:** 82% · **Complexity:** Medium · **Status:** Unexplored

### F. "Lab notebook, not autosave" — the differentiation narrative
**Axis:** Differentiation narrative
**Description:** The pitch vs Anthropic Dreaming / Honcho / Mem0, anchored in the governing
doctrine (Zhang et al.): rivals treat consolidation as *autosave* (silent, in-place,
trust-the-model); hypermnesic treats it as a *lab notebook* — dated, cited, peer-reviewed-
before-publish, never overwritten. Provenance lives in the ledger; source notes stay byte-clean.
Headline: "a dream that left no cited, committed, reviewable trail did not happen." Reframes the
v0 ephemeral/pull-only posture as a principled stance, not a missing feature.
**Basis:** `external:` Zhang et al. 2605.12978 (consolidation degrades 46–54% of solved
problems), Mem0 add-only pivot, Anthropic review-before-write, Belief-Aware Memory. `direct:`
doctrine "NOT an autonomous re-organizer — no silent rewrites, EVER."
**Rationale:** Fuses all three ideas into one defensible sentence and a moat — git is the
"anchor" the skeptic literature says every consolidator lacks.
**Downsides:** Positioning, not mechanism — needs held-out numbers (parent idea #6) to fully
back the claim.
**Confidence:** 84% · **Complexity:** Low · **Status:** Unexplored
